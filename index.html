Apresentação Fabrica de Software Opet - FSO


A Fabrica de software Opet - FSO se destina a todos os alunos interessados em consolidar conhecimentos adquiridos e coloca-los em prática no desenvolvimento de sistemas. Busca proporcionar experiência semelhante a uma fabrica de software comercial. Os alunos terão a oportunidade de atuar nos diversos papeis que encontrarão como profissionais de TI.(Front-end, Back-end, DBA, PM, Web Designer, etc). Os alunos mais antigos atuam como monitores dos iniciantes.

A FSO conta com o apoio de professores que ajudam o desenvolvimento esclarecendo dificuldades técnicas. Para atuar na FSO o aluno além de interesse deve ter disponibilidade para se dedicar às atividades propostas. Será realizado um cadastro preliminar do aluno para inicio das atividades.

Antes de preencher e assinar este cadastro o aluno deve se familiarizar com o ritmo de trabalho da FSO e escolher um grupo de trabalho (ex. Front-end). O aluno receberá atividades coerentes com seu nível de conhecimento e essas atividades deverão ser executadas no primeiro mês. A cada atividade corresponde um numero de horas que serão contabilizadas como horas complementares ao curso TDS, TGI ou outro segundo as regras definidas abaixo. Ao final deste período o cadastro preliminar será efetivado e o aluno terá ingressado na FSO. Os alunos que desistirem durante o período preliminar não terão as horas trabalhadas contabilizadas como horas complementares. Já os alunos efetivados terão as horas trabalhadas no primeiro mês contabilizadas como horas complementares.

Tecnologias Utilizadas na FSO

developer_mode

Front-end

"O desenvolvedor front-end é responsável por "dar vida" à interface. Trabalha com a parte da aplicação que interage diretamente com o usuário. Por isso, é importante que esse desenvolvedor também se preocupe com a experiência do usuário. Na parte de estudo, este profissional foca em HTML (linguagem de marcação), CSS (linguagem de estilo) e JavaScript (linguagem de script/programação)."

Tecnologias Utilizadas
  • HTML5 + CSS3
  • Bootstrap 3.3.7 / 4.0
  • JavaScript
  • jQuery

code

Back-end

"Como o nome sugere, o desenvolvedor back-end trabalha na parte de "trás" da aplicação. Ele é o responsável, em termos gerais, pela implementação da regra de negócio. Em uma aplicação web, este desenvolvedor, quando focado, não toca na parte visual da aplicação. Por lidar com a regra de negócio, as vezes um programador de sistemas, como de aplicações comerciais e até científicas, pode ser chamado de desenvolvedor back-end. E geralmente, nessas aplicações, este desenvolvedor trabalha um pouco com a parte visual."

Tecnologias Utilizadas
  • Java Orientado a Objetos
  • Java Spring
  • Python / Django / DjangoREST

storage

Banco de Dados

“Bancos de dados, ou bases de dados (em Portugal), são coleções de informações que se relacionam de forma que crie um sentido. São de vital importância para empresas, e há duas décadas se tornaram a principal peça dos sistemas de informação.“

Tecnologias Utilizadas
  • Oracle
  • MYSQL
  • SQLite 3
  • PostgreSQL

Horas Complementares

Executando atividades e participando da Fabrica de Software é contabilizado para as Horas complementares. Sendo possivel realizar as atividades via Trabalho Remoto ou Presencial, porem cada uma das formas é contabilizado de maneira difirente as Horas. Abaixo segue a explicação de cada uma das formas.

access_alarm

Presencial

Para as atividades realizadas presencialmente na Opet, equivale a 1/8 de hora complementar.

Exemplo: 4 meses de participação = 4h x 5 dias por semana x 4 semanas x 4 meses
